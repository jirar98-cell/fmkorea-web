### 🔑 핵심 방향 전환 (제일 중요 — 다음 작업이 다 여기서 나옴)
이 앱의 본질 = '발견 도구'가 아니라 **'발견 + 판단 자동화 도구'**.
사용자의 진짜 피로 = 소재 발견이 아니라 **"이게 뜰/쓸 자료인지 판단"하는 시간**.
→ **AI 채점(=판단 엔진)을 사용자의 실제 판단기준대로** 만드는 게 앞으로의 핵심.

**사용자 판단기준 3개 (채점에 반영해야 함):**
 - ① **인플루언서 개인 저작물이 아닐 것** — 현재 채점에 **없음, 추가 필요**
 - ② **이미 타 커뮤니티에서 검증된 반응** ("알았지만 몰랐던" 류 포함) — 추천수로 일부 반영 중
 - ③ **본인도 재밌고 + 남들이 많이 안 쓴 희소 소스** — 현재 채점에 **없음, 추가 필요**

**범위 경계:**
 - 저작권 판단은 앱에서 **제외** (제작 단계에서 사용자가 직접 출처 보고 판단).
 - 제작은 AI에 안 맡김 (사용자 영역). **발견+판단까지만** 자동화 지향.

### 남은 작업 (우선순위)
1. **채점 기준에 ①개인저작물 아님 + ③희소성 추가** — 판단 엔진을 사용자 기준대로 만드는 핵심. `shorts_filter.py`의 `_CHANNEL_DNA` 채점 기준을 손봐야. ③희소성은 추천수 외 신호(소스 분포 등)가 필요할 수 있어 설계 논의 선행.
2. **채점 실패(점수 배지 없는 잡학 더미) 해결** — `app.py` fallback: AI 실패 시 잡학으로 버리지 말고 `_guess_tags`(app.py:301)로 키워드 추측이라도 부여 + **rate-limit 완화**(콜드로드 ~310개 한번에 던지는 것 축소 — `_ai_filter_posts` CONCURRENT/BATCH, 소스별 페이지수). 무료티어(8b) 유지.
3. **소스 적합성 재검토** — FMKorea 등 텍스트 커뮤니티 소스가 영상훅 중심 채널 소재풀과 안 맞을 가능성. 근본적이나 무거워서 후순위.

### 제약

### 내일 할 것 — 사용자가 강하게 원함 (우선순위 1)
**피드백 루프 = 하트 → AI 취향 학습 → 피드 우선순위 반영**
- 현재 하트(❤)는 저장만 되고 학습에 안 쓰임. 이걸 실제 학습에 연결하는 게 핵심.
- 설계:
  1. 하트 누른 게시물 제목들을 모음 (이미 localStorage/likes에 있음 — 위치 확인 필요)
  2. AI가 그 제목들의 공통 패턴을 **언어로** 뽑음 ("동물이 사람처럼 행동 + 숫자 반전 선호")
  3. 그 패턴을 `shorts_filter.py` 채점 프롬프트에 주입 → 매칭되는 소재 점수 가산
  4. 누를수록 정확해짐
- **B안(말로 지시)**: 텍스트칸에 "요즘은 인물 인생역전이 땡겨" 입력 → 채점 기준 실시간 반영. 프롬프트에 "사용자 현재 취향" 한 줄 끼우면 됨. ~30분.
- 사용자 비전: 이 루프 구조를 식당/테이블오더 관찰/메뉴기획 등 **잡다한 일 자동화**에 확장하려는 의도. "좋은 예시 모음 → AI가 패턴 언어화 → 다음 생성" 뼈대.
