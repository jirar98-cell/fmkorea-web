## 마지막 세션: 2026-06-12 (밤2, 레딧 시도 + 피드백루프 설계 논의)

### 이번 추가 세션 한 것
- **레딧 소스 추가 시도 → Railway에서 차단 확인**
  - 채널 DNA 최적 14개 서브레딧 큐레이션 (Awwducational/todayilearned/AnimalsBeingBros/NatureIsFuckingLit/nextfuckinglevel 등)
  - Reddit `.json`은 OAuth 봉쇄(403), **`.rss`는 로컬 주거용 IP에선 200** 으로 됨 (html.parser 파싱, lxml 불필요)
  - **그러나 Railway 데이터센터 IP는 RSS도 403** → 0개. 코드 문제 아님, IP 문제.
  - 결론: 레딧을 피드/프리웜에서 **제외**. 함수(`_fetch_reddit_*`)는 보존 — 나중에 OAuth(client id/secret) 또는 프록시 붙이면 부활 가능. `/api/feed/source?src=reddit` 로는 여전히 호출됨.
  - BP 번역 로직을 공용 함수 `_translate_score_posts(posts, deadline)` 로 추출 (BP/레딧 공용)
- XMLParsedAsHTMLWarning 억제 추가

### 내일 할 것 — 사용자가 강하게 원함 (우선순위 1)
**피드백 루프 = 하트 → AI 취향 학습 → 피드 우선순위 반영**
- 현재 하트(❤)는 저장만 되고 학습에 안 쓰임. 이걸 실제 학습에 연결하는 게 핵심.
- 설계:
  1. 하트 누른 게시물 제목들을 모음 (이미 localStorage/likes에 있음 — 위치 확인 필요)
  2. AI가 그 제목들의 공통 패턴을 **언어로** 뽑음 ("동물이 사람처럼 행동 + 숫자 반전 선호")
  3. 그 패턴을 `shorts_filter.py` 채점 프롬프트에 주입 → 매칭되는 소재 점수 가산
  4. 누를수록 정확해짐
- **B안(말로 지시)**: 텍스트칸에 "요즘은 인물 인생역전이 땡겨" 입력 → 채점 기준 실시간 반영. 프롬프트에 "사용자 현재 취향" 한 줄 끼우면 됨. ~30분.
- 사용자 비전: 이 루프 구조를 식당/테이블오더 관찰/메뉴기획 등 **잡다한 일 자동화**에 확장하려는 의도. "좋은 예시 모음 → AI가 패턴 언어화 → 다음 생성" 뼈대.

### 내일 할 것 — 기술 부채 (우선순위 2)
- **콜드스타트 피드 타임아웃**: 배포 직후~프리웜 완료(~2분) 사이 `/api/feed`가 동기 페치로 60~90s 블로킹/타임아웃.
  - 원인: `get_cached`의 콜드패스(캐시 완전 비었을 때)가 `_run_async()`로 동기 블로킹. 프리웜이 단일 `_loop` 점유 시 요청이 큐잉됨.
  - 해결안: 콜드패스를 non-blocking으로 — 빈 캐시면 백그라운드 갱신 트리거하고 즉시 `([], "로딩중", 0)` 반환. 프론트가 폴링/자동새로고침으로 채움. (프론트 로딩바가 이미 진행형 UI라 호환될 듯, 동작 확인 필요)
- 인스티즈 간헐 0개 / 개드립 2~9개 — 셀렉터 점검

### 미완료 측정
- 종합 테스트(영문 강등 실제 작동 여부, 태그/점수 분포) 백그라운드 돌리다 세션 종료. 내일 `/api/feed` 떠 있으면 재측정.
